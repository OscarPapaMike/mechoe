mdata CLI

Scryfall data manager. Downloads card JSON and art, builds a local SQLite search index, and caches everything under data/ so the rest of the toolchain can work offline.

First-time setup: run mdata sync once to build the local index before any other command will work. The bulk download is ~540 MB but is removed after indexing; the index itself is much smaller.

Building & running

# from mechoe/mdata/
~/.cargo/bin/cargo build --release
./target/release/mdata <command> [args]

# or during development
~/.cargo/bin/cargo run -- <command> [args]

The data directory is resolved automatically by walking up from the current working directory looking for data/_meta/. You can override it:

mdata --data-dir /path/to/data <command>
# or
MECHOE_DATA=/path/to/data mdata <command>

Commands

CommandDescription
sync Download the Scryfall bulk card file, build data/_meta/index.db, and sync mana symbol SVGs. Must be run at least once.
fetch "<name>" Fetch a card by exact name (case-insensitive). Downloads the oldest printing's JSON and art crop into data/<SET>/. Skips download if already cached.
get <SET> <num> Fetch a specific printing by set code and collector number.
pull <SET> Download all cards in a set (JSON + art). Skips files already cached.
info "<name>" List all printings of a card with cache status. No files downloaded.
fetch "<name>" --info-only Same as info.

Examples

# First-time setup
mdata sync

# Get a card (oldest printing)
mdata fetch "Lightning Bolt"

# See all printings without downloading
mdata info "Lightning Bolt"

# Get a specific printing (Alpha Shivan Dragon)
mdata get LEA 175

# Download an entire set
mdata pull DRK

Data layout

data/
├── _meta/
│   ├── index.db          # SQLite search index (built by sync)
│   ├── symbols/          # Mana symbol SVGs (R.svg, G.svg, ...)
│   └── rails/            # Frame decoration assets
├── LEA/
│   ├── 1.json            # Card JSON (Scryfall format)
│   └── 1.jpg             # Art crop
└── DRK/
    ├── 1.json
    └── 1.jpg

mart CLI

MTG proxy card renderer. Takes a Scryfall-format JSON file and an optional art image and produces a high-resolution PNG card in a 1993-era premodern frame style.

Building & running

# from mechoe/mart/
~/.cargo/bin/cargo build --release
./target/release/mart <command> [args]

# or during development
~/.cargo/bin/cargo run -- <command> [args]

Symbols and font assets are resolved automatically by walking up from the current directory. Override with flags or MECHOE_DATA.

Commands

render

Render a single card to PNG.

mart render <card.json> [art.jpg] [options]
card.json
Path to a Scryfall-format card JSON file (from mdata fetch or manually written).
art.jpg
Optional artwork file (PNG or JPEG). If omitted a placeholder fills the art box.
-o, --out <path>
Output PNG path. Default: out.png.
--dpi <n>
Render resolution. Default: 300 (750×1050 px). Use 1200 for print-quality output.
--style basic|classic
Frame style. basic (default) is the flat two-color premodern look. classic uses explicit frame rails.
--fonts-dir <dir>
Directory containing Windsor font files if not installed system-wide.
--symbols-dir <dir>
Directory containing mana symbol SVGs. Auto-resolved from data/_meta/symbols/.

gen-symbols

Generate SVG files for generic mana numerals (0–16, X, T) using Windsor Heavy. Run this after changing the font or numeral styling.

mart gen-symbols [--out-dir <dir>] [--fonts-dir <dir>]

Output defaults to data/_meta/symbols/.

Examples

# Render at 300 DPI with placeholder art
mart render data/LEA/1.json -o /tmp/ancestral_recall.png

# Render with art at print quality
mart render data/LEA/1.json data/LEA/1.jpg -o /tmp/ancestral_recall.png --dpi 1200

# Render a card fetched by mdata (art + JSON)
mdata fetch "Birds of Paradise"
mart render data/LEA/35.json data/LEA/35.jpg -o /tmp/bop.png --dpi 600

# Regenerate numeral symbols after font change
mart gen-symbols

Card JSON format

mart reads standard Scryfall card JSON. Key fields used:

FieldUsed for
nameTitle bar text
mana_costMana symbols in title bar (e.g. {2}{G}{G})
type_lineType line text and frame color logic
oracle_textRules text body
flavor_textItalic flavor text (supports *title* for upright spans)
power / toughnessP/T in bottom-right corner (creatures only)
colorsFrame color selection
set / collector_numberSet code and number stamp in type line

mgui GUI

Live card preview. A desktop window that watches a card JSON file and re-renders the card automatically whenever the file changes. Useful for iterating on card designs without re-running the CLI.

Building & running

# from mechoe/mgui/
~/.cargo/bin/cargo run --release

Usage

mgui opens a window with a file picker at the top. Enter the path to a Scryfall JSON file and an optional art file. The card re-renders automatically when either file is modified on disk.

The preview respects the same rendering pipeline as mart render — what you see in mgui is what you get from mart.

Tips

  • Use mdata fetch to get a card JSON, then open it in mgui while editing.
  • mgui renders at screen resolution; use mart render --dpi 1200 for the final file.

mtable GUI

Virtual playmat. A desktop application for arranging and playing with cards on a zoomable, pannable board. Supports tapping, counters, grid snapping, and board save/load.

Building & running

# from mechoe/mtable/
~/.cargo/bin/cargo run --release

InputAction

Grid snapping

Cards snap to a 100×140 grid when dropped within 40×60 px of a grid intersection. A semi-transparent red overlay shows the snap target while dragging. Counters do not snap.

Save & load

The Save button writes the full board state (cards, counters, positions, artwork, z-order) to mtable_save.json in the working directory. Load restores it.

Typical workflow

  1. Run mdata sync once to build the local index (requires internet).
  2. Use mdata fetch "Card Name" to download a card's JSON and art into data/.
  3. Open the JSON in mgui for a live preview while tweaking.
  4. Run mart render data/SET/NUM.json data/SET/NUM.jpg -o out.png --dpi 1200 to produce the final print-quality PNG.
  5. Load finished cards into mtable to play with them on a virtual board.
